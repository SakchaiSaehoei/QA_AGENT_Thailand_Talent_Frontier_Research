Slide 7: Core Idea 4 — RAG
Purpose

Explain Retrieval-Augmented Generation in a simple way.

Bullet points

RAG stands for Retrieval-Augmented Generation

It means: retrieve information first, then generate an answer

The model does not memorize everything

RAG lets the model use external documents

Jump to code?

No

Speaker Script

RAG is a very important idea.

Instead of asking the model to “remember everything,” we let it look up information first.

First, we retrieve relevant documents.

Then, we give those documents to the LLM.

Finally, the LLM generates an answer based on that information.

So RAG = Search first → Answer second.

This is much safer and more accurate than relying only on the model’s memory.

Slide 8: Core Idea 5 — Database
Purpose

Explain the role of databases in AI systems.

Bullet points

A database stores structured data

Examples: sales records, student data, inventory

LLMs do not automatically know database content

Agents can query databases using tools

Jump to code?

No

Speaker Script

A database is where we store structured information.

For example: sales numbers, product lists, student grades.

The LLM does not magically know what is inside our company database.

So we must connect it using tools.

The agent can translate a question into a database query.

Then it uses the results to generate a final answer.

This is how we build real-world AI systems.
